# FE5 风格战棋模块划分文档

## 1. 文档目标

本文件用于规划《多拉基亚 776》（FE5）风格战棋复刻项目的功能模块边界。它不是逐条公式实现文档，而是用于回答三个问题：

- 游戏由哪些主要系统组成。
- 每个系统负责什么，不负责什么。
- 各系统之间如何协作，应该按什么顺序开发。

项目当前使用 Godot 4.x + C#，已有 `Combat`、`Core`、`Map`、`Units`、`Test` 等目录。本模块划分会兼容现有结构，同时为后续扩展预留数据、UI、章节事件、AI、存档等系统。

## 2. 总体模块图

```text
Game
├── Core                  游戏流程、状态机、回合推进
├── Chapter               章节规则、胜败条件、事件触发
├── Map                   地图格子、地形、寻路、范围计算
├── Unit                  单位、职业、属性、状态、疲劳
├── Item                  武器、道具、耐久、交换、仓库
├── Combat                战斗预测、命中、伤害、追击、特效
├── Action                移动、攻击、救出、捕获、访问、等待等行动
├── AI                    敌军行为、目标选择、行动评分
├── UI                    战棋操作界面、菜单、预测窗口、状态面板
├── Data                  静态数据表、配置加载、资源索引
├── Save                  战役进度、章节中断、设置保存
├── Presentation          动画、音效、镜头、战斗表现
└── Test                  规则测试、集成测试、回归验证
```

## 3. 核心开发原则

- 规则优先于表现：先保证移动、战斗、行动结算正确，再做动画和镜头。
- 数据驱动：单位、职业、武器、地形、章节事件尽量从资源或表格加载，避免写死在节点脚本里。
- 纯逻辑可测试：命中、伤害、移动范围、AI 评分等核心逻辑应尽量脱离 Godot 场景节点，方便单元测试。
- 状态统一收口：回合、当前单位、选择模式、行动结果等由核心流程管理，避免 UI 和地图脚本各自维护一套状态。
- FE5 特性单独建模：捕获、救出、疲劳、再行动、体格、偷窃、俘虏、撤退等机制不要混入普通攻击逻辑。

## 4. 模块详述

### 4.1 Core：游戏流程模块

负责游戏运行时的最高层状态管理。

主要职责：

- 管理游戏状态：标题、章节准备、战斗地图、战斗结算、存档读档等。
- 管理回合状态：玩家回合、敌军回合、NPC 回合、章节事件阶段。
- 控制输入是否可用，避免动画或结算中重复操作。
- 统一派发“回合开始”“单位行动结束”“章节结束”等事件。

不负责：

- 不计算战斗公式。
- 不直接渲染地图。
- 不保存具体单位属性表。

建议类：

- `GameManager`
- `TurnManager`
- `GameState`
- `TurnPhase`
- `CommandContext`

### 4.2 Chapter：章节与事件模块

负责一章战斗的整体规则和脚本事件。

主要职责：

- 加载章节地图、初始单位、敌军配置、胜败条件。
- 管理章节目标：制压、逃脱、防守、击破 Boss、全灭等。
- 管理事件触发：开场、回合数、区域进入、单位死亡、访问村庄、对话、增援。
- 管理章节内特殊规则：撤退点、门、桥、宝箱、村庄、俘虏释放等。

FE5 相关重点：

- 逃脱章节中，主角撤退后可能导致未撤退友军被俘。
- 增援和事件常常与回合数、区域、敌将状态绑定。
- 部分章节有俘虏、NPC、绿色单位、村庄奖励等复杂目标。

建议数据：

- `ChapterDefinition`
- `ChapterObjective`
- `ChapterEvent`
- `ReinforcementGroup`
- `DeploymentSlot`

### 4.3 Map：地图与格子模块

负责网格地图、地形、占位和移动范围。

主要职责：

- 管理地图尺寸、格子坐标、地形类型。
- 查询地形效果：移动消耗、防御、回避、回复、是否可通行。
- 维护单位占位：某格是否有单位、阵营、是否可穿越。
- 计算移动范围、攻击范围、危险范围。
- 提供寻路结果给 UI 和行动模块。

不负责：

- 不决定敌人想攻击谁。
- 不结算战斗伤害。
- 不直接改变单位生命值。

建议类：

- `MapManager`
- `Tile`
- `TerrainDefinition`
- `Pathfinder`
- `RangeCalculator`
- `OccupancyMap`

### 4.4 Unit：单位、职业与成长模块

负责所有可行动角色的基础数据和临时状态。

主要职责：

- 保存单位身份：角色 ID、名字、阵营、职业、等级、经验。
- 保存战斗属性：HP、力量、魔力、技术、速度、幸运、防守、体格、移动。
- 管理成长率、升级、职业上限、晋级。
- 管理临时状态：中毒、睡眠、沉默、混乱、疲劳、已行动、被救出等。
- 管理支援、个人技能、再行动星等扩展能力。

FE5 相关重点：

- 魔力同时承担魔法攻击和魔法防御。
- 体格影响攻速、救出、捕获、偷窃等机制。
- 疲劳会影响下一章是否可出击。
- 再行动星可能让单位行动后再次获得行动机会。

建议类：

- `Unit`
- `UnitStats`
- `ClassDefinition`
- `GrowthManager`
- `StatusEffect`
- `FatigueSystem`

### 4.5 Item：物品、武器与库存模块

负责武器、道具、耐久和交换。

主要职责：

- 管理单位背包、仓库、掉落物、宝箱物品。
- 定义武器属性：威力、命中、重量、必杀、射程、耐久、武器等级。
- 定义武器类型：剑、枪、斧、弓、理、光、暗、杖等。
- 处理使用、交换、丢弃、获得、修理、耐久消耗。
- 提供可装备、可攻击、可使用的判定。

FE5 相关重点：

- 武器耐久非常关键，应作为战斗结算的一部分。
- 捕获敌人后可夺取敌方物品。
- 杖系统是重要战术核心，需要单独处理命中、经验和效果。

建议类：

- `ItemInstance`
- `ItemDefinition`
- `WeaponDefinition`
- `Inventory`
- `TradeService`
- `DurabilityService`

### 4.6 Combat：战斗计算模块

负责从攻击声明到战斗结果的完整规则计算。

主要职责：

- 生成战斗预测：伤害、命中、必杀、攻速、追击。
- 结算命中、闪避、必杀、特效、武器克制。
- 处理追击、反击、勇者武器、多段攻击。
- 处理战斗后经验、武器经验、耐久、死亡判定。
- 输出结构化战斗结果，供 UI、动画、日志和测试使用。

不负责：

- 不移动单位。
- 不弹出战斗菜单。
- 不播放动画。

FE5 相关重点：

- 追击阈值通常为攻速差 4。
- 武器重量通过体格影响攻速。
- 捕获状态、救出状态会影响战斗能力。
- 部分武器有特效、异常状态、吸血或特殊脚本效果。

建议类：

- `CombatCalculator`
- `BattleManager`
- `BattleForecast`
- `BattleResult`
- `AttackSequence`
- `ExperienceManager`

### 4.7 Action：单位行动模块

负责“单位选择一个命令并执行”的完整流程。

主要职责：

- 管理行动类型：移动、攻击、杖、物品、交换、救出、放下、捕获、偷窃、访问、开门、等待。
- 校验行动是否合法。
- 将地图、单位、物品、战斗等模块串联成一次可回滚或可记录的行动。
- 输出行动结果，通知 Core 结束单位行动或触发再行动。

FE5 相关重点：

- 捕获不是普通攻击的附属品，应作为独立行动入口。
- 救出、交接、放下会改变两个单位的位置和状态。
- 偷窃需要体格、速度、物品类型等判定。
- 访问、开门、宝箱可能触发章节事件。

建议类：

- `ActionManager`
- `UnitAction`
- `ActionValidator`
- `ActionResult`
- `RescueService`
- `CaptureService`
- `StealService`

### 4.8 AI：敌军与 NPC 行为模块

负责非玩家阵营的行动决策。

主要职责：

- 根据 AI 配置选择行动：进攻、防守、逃跑、治疗、开门、护送等。
- 评估目标优先级：击杀、伤害、命中、危险、地形、保护目标。
- 计算敌方移动后可攻击范围。
- 支持章节脚本强制行为，例如增援移动路线、Boss 不动、盗贼抢宝。

建议阶段：

- 第一阶段：最近目标 + 可攻击就攻击。
- 第二阶段：伤害/击杀评分 + 生存风险。
- 第三阶段：章节脚本 AI、盗贼 AI、治疗 AI、撤退 AI。

建议类：

- `AiController`
- `AiProfile`
- `AiDecision`
- `TargetEvaluator`
- `ThreatMap`

### 4.9 UI：战棋操作界面模块

负责玩家可见和可操作的界面。

主要职责：

- 地图光标、单位选择、移动路径预览。
- 行动菜单、武器选择、物品菜单、交换界面。
- 战斗预测窗口、单位状态窗口、地形信息窗口。
- 回合提示、章节目标、对话框、系统菜单。
- 鼠标、键盘、手柄输入映射。

设计重点：

- UI 只发出“玩家意图”，不要直接修改战斗数据。
- 预测窗口必须来自 Combat 和 Action 的正式计算，避免显示与实际结算不一致。
- 地图上的范围显示由 Map 模块提供结果，UI 只负责表现。

建议类：

- `InputController`
- `CursorController`
- `CommandMenu`
- `ForecastPanel`
- `UnitStatusPanel`
- `DialogueBox`

### 4.10 Data：数据与配置模块

负责所有静态配置的加载和索引。

主要职责：

- 加载角色、职业、武器、地形、章节、对话、动画配置。
- 提供 ID 到定义对象的查询。
- 校验数据完整性，例如武器引用了不存在的类型。
- 支持后续编辑器工具或表格导入。

建议数据来源：

- Godot `Resource`
- JSON
- CSV
- 自定义编辑器工具

建议类：

- `DataRegistry`
- `UnitDefinition`
- `ClassDefinition`
- `ItemDefinition`
- `TerrainDefinition`
- `ChapterDefinition`

### 4.11 Save：存档模块

负责长期进度和章节中断。

主要职责：

- 保存战役进度：已通关章节、角色成长、死亡、疲劳、仓库、金钱。
- 保存设置：音量、显示速度、输入配置。
- 支持章节中断存档：当前回合、地图单位状态、事件状态。
- 提供版本号和迁移策略，避免数据结构变化后旧存档完全不可用。

FE5 相关重点：

- 永久死亡需要被战役存档记录。
- 疲劳、俘虏、逃脱状态会影响下一章。
- 章节中断存档应尽量完整保存随机数状态，保证读档后行为可复现。

建议类：

- `SaveManager`
- `CampaignSave`
- `ChapterSave`
- `SaveVersion`

### 4.12 Presentation：表现层模块

负责视觉、音效和节奏。

主要职责：

- 地图单位动画、移动动画、攻击动画。
- 战斗演出、特效、音效、屏幕震动。
- 镜头移动、聚焦、过场演出。
- 章节标题、回合横幅、胜败演出。

开发原则：

- 表现层消费 Action 和 Combat 的结果，不反向决定规则。
- 动画可以延迟显示结算，但规则结果应先由逻辑层确定。
- 早期可用简单日志和占位动画替代。

建议类：

- `AnimationDirector`
- `CameraController`
- `BattleAnimationPlayer`
- `SfxManager`
- `VfxSpawner`

### 4.13 Test：测试与验证模块

负责验证核心规则不会回归。

主要职责：

- 单元测试：战斗公式、成长、移动范围、物品判定。
- 集成测试：一次完整攻击、捕获、救出、交换、章节胜利。
- 数据校验：所有章节引用的数据都存在，所有武器数值合法。
- 回归用例：FE5 关键机制固定输入得到固定输出。

优先测试点：

- 攻速计算。
- 追击判定。
- 物理/魔法伤害。
- 命中和必杀范围限制。
- 移动消耗和不可通行地形。
- 捕获、救出、偷窃的合法性判定。

## 6. 开发里程碑

### Milestone 1：最小可玩战棋循环

目标：可以在一张测试地图上选择单位、移动、攻击、结束回合。

包含模块：

- Core：回合切换。
- Map：格子、地形、移动范围。
- Unit：基础属性。
- Combat：基础伤害、命中、死亡。
- Action：移动、攻击、等待。
- UI：光标、移动范围、行动菜单。

### Milestone 2：FE5 基础战斗味道

目标：战斗规则接近 FE5 的核心手感。

包含功能：

- 体格与武器重量影响攻速。
- 追击、反击、必杀。
- 武器耐久和武器类型。
- 经验、升级、成长。
- 地形防御和回避。

### Milestone 3：FE5 特色行动

目标：出现区别于普通战棋的 FE5 特性。

包含功能：

- 救出、交接、放下。
- 捕获、夺取物品。
- 偷窃。
- 疲劳。
- 杖系统初版。

### Milestone 4：章节系统

目标：能制作一章有事件、有胜败条件的关卡。

包含功能：

- 章节数据加载。
- 敌军和 NPC 配置。
- 胜败条件。
- 对话、访问、增援。
- 撤退点或制压点。

### Milestone 5：战役流程

目标：多章节连续游玩。

包含功能：

- 战役存档。
- 章节准备。
- 仓库、金钱、出击选择。
- 死亡、疲劳、俘虏状态跨章节继承。

## 7. 模块依赖建议

```text
Data -> Unit / Item / Map / Chapter
Map -> Unit
Combat -> Unit / Item / Map
Action -> Unit / Item / Map / Combat / Chapter
AI -> Action / Map / Unit / Combat
UI -> Action / Map / Unit / Combat
Core -> Chapter / Action / AI / UI
Save -> Core / Chapter / Unit / Item / Map
Presentation -> Action / Combat / UI
Test -> all logic modules
```

依赖方向建议保持为“上层编排调用下层规则”。例如 UI 可以请求 Action 执行攻击，但 Combat 不应该反过来依赖 UI。

## 8. 下一步文档拆分建议

后续可以按以下顺序继续写详细设计：

1. `Combat_System_Design.md`：战斗公式、攻击序列、预测与实际结算。
2. `Action_System_Design.md`：移动、攻击、救出、捕获、偷窃等命令流程。
3. `Data_Schema_Design.md`：单位、职业、武器、地形、章节数据格式。
4. `Chapter_Event_Design.md`：事件触发器、条件、结果、对话和增援。
5. `Save_System_Design.md`：战役存档和章节中断存档结构。
